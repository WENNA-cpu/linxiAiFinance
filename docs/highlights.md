# 灵析 AI 智能投顾助手 — 核心亮点总结

> 版本：v1.4 · 更新日期：2026-06-23  
> 适用场景：面试开场 / 作品集首页 / 1 分钟电梯演讲

---

## 亮点一：合规闭环设计

### 问题

AI 理财工具最大的风险不是「不准」，而是「越界」——荐股、承诺收益、诱导买卖。监管和用户都要求：**能拦截、能留痕、能统计**。

### 灵析方案

```
用户输入 → 规则引擎前置校验 → 通过才调用 LLM → 拦截写入 compliance_logs → 合规面板统计展示
```

| 环节 | 实现 | 文件 |
| ---- | ---- | ---- |
| 禁止词库 | 11 个关键词（荐股、能买吗、保证收益…） | `rule_engine.py` |
| 前置拦截 | 诊断追问 / 规则检查 API 先过规则引擎 | `diagnose_chat.py`, `rule.py` |
| 拦截留痕 | 写入 `compliance_logs` + `audit_logs` | `compliance_log_service.py` |
| 统计展示 | 拦截次数、幻觉修正率、规则通过率 — **基于系统运行日志的实时统计** | `admin.py` → `Compliance.vue` |
| 交互兜底 | 风险提示弹窗 + 常驻免责声明 | `RiskBanner.vue`, `Compliance.vue` |

### 面试一句话

> 「我们不做『事后补救』，而是把规则引擎放在 LLM 调用之前——用户问『茅台能买吗』，系统在 10ms 内拦截，写入合规日志，前端展示拦截原因，全程零 LLM 调用、零合规风险。」

## 亮点二：诊断追问助手（RAG + 合规 + DeepSeek）

### 问题

诊断报告生成后，用户还有追问——「我的茅台亏了多少」「宁德时代怎么样」。需要**基于持仓上下文**回答，同时**不能越界荐股**。

### 灵析方案

| 能力 | 实现 |
| ---- | ---- |
| **位置** | 持仓诊断结果页 **底部** |
| **RAG 上下文** | 诊断结果结构化注入 Prompt（资产、成本、现价、盈亏、风险） |
| **三种模式** | portfolio（持仓内）/ external（持仓外）/ general（整体） |
| **合规前置** | 禁止词拦截 → 403，零 LLM |
| **DeepSeek** | 规则放行后生成合规回答 |

### 差异化

| 场景 | 回答策略 |
| ---- | -------- |
| 持仓内「我的茅台怎么样」 | 带成本价、现价、盈亏、风险等级 |
| 持仓外「宁德时代怎么样」 | 通用介绍 + 免责声明，禁止买卖建议 |
| 违规「能买吗」 | 直接拦截，不调 LLM |

### 面试一句话

> 「追问助手不是通用 ChatGPT——它把诊断结果当 RAG 上下文注入，持仓内的股票能算盈亏，持仓外的只做科普介绍，违规问题规则引擎直接拦。」

## 亮点三：双模型协同 — LSTM v2.0 + 随机森林 v2.2

### 双模型分工

| 模型 | 版本 | 任务 | 输入 | 核心指标 | 文件 |
| ---- | ---- | ---- | ---- | -------- | ---- |
| **LSTM** | **v2.0** | 周期/PE 趋势参考 | 30 日 × **7 特征** | RMSE **3.39**，MAE **2.52** | `lstm_cycle.h5` |
| **随机森林** | **v2.2** | 风险三分类 | **14 维**（含 3 项交互特征） | 准确率 **61.17%**，F1 **61.19%** | `rf_risk.pkl` |

### LSTM v2.0 七特征

```
close · pct_chg · vol · pe_pct · pb_pct · turnover_rate · amplitude
```

估值数据 **AKShare 优先、Tushare 降级**；300 股 × 3 年训练。

### 随机森林 v2.2 迭代路径

| 版本 | 准确率 | 主要变更 |
| ---- | ------ | -------- |
| V1 | 46.16% | 4 维基线 |
| V2 | 54.82% | +PE/PB 分位、振幅 |
| V2.1 | 56.91% | +RSI、MA 乖离、布林带、量比 |
| **V2.2** | **61.17%** | +交互特征（PE×波动率等）+ 超参调优 |

**Top 5 特征重要性：** PB 分位、PE 分位、**PE×波动率**、波动率、布林带宽度

### 工程化训练流程

```
data_fetch.py → feature_engineering.py → train_lstm_v2.py / train_rf.py → model_manager 注册 → HTTP 回滚
```

### 面试一句话

> 「LSTM v2.0 七特征 RMSE 3.39；随机森林从 46% 迭代到 v2.2 **61%**，靠特征工程 + 交互项 + 调参；模型未加载时规则引擎兜底，且支持 HTTP 热回滚。」

## 亮点四：模型版本管理与 HTTP 回滚

### 问题

模型上线后如果效果变差，需要**快速回退**，且**不能停机**。

### 灵析方案

```
训练完成 → register_model_version() → models/versions/ 归档
                ↓
异常发现 → POST /api/admin/model/rollback → 恢复文件 → 热重载 → 下次推理用旧模型
```

| 能力 | 实现 | 说明 |
| ---- | ---- | ---- |
| 版本注册 | `register_model_version()` | 训练后自动归档 + 写 metadata |
| 元数据 | `model_metadata.json` | active + history（最多 20 条） |
| HTTP 回滚 | `POST /api/admin/model/rollback` | 支持 `v1.0` 语义别名 |
| 热重载 | `reload_lstm_predictors()` / `reload_rf_predictor()` | 清空单例，**不重启 Uvicorn** |
| 灰度 | `NEW_MODEL_RATIO=5` | 5% 流量验证新模型 |

### 面试一句话

> 「从训练注册到灰度到 HTTP 回滚，我们有一套 MLOps 最小闭环——回滚接口复制归档文件、更新 runtime 配置、热重载推理缓存，全程不需要重启服务。」

## 亮点五：真实持仓情绪纠偏

### 问题

行为金融学概念很多，但如果没有**用户自己的持仓数据**，就是空泛说教。

### 灵析方案

| 偏差类型 | 计算逻辑 | 数据来源 |
| -------- | -------- | -------- |
| **确认偏误** | 最高行业市值占比 | 真实持仓 + Tushare 行业 |
| **损失厌恶** | 亏损资产数量 | costPrice vs currentPrice |
| **羊群效应** | 行业集中度 | 最高行业 >50% 预警 |
| **行为矫正估算** | 基于持仓风险系数 | 基于真实持仓的动态估算 |

### 面试一句话

> 「情绪指数基于 Tushare 舆情数据实时计算；确认偏误、损失厌恶、羊群效应均基于真实持仓动态计算；行为矫正估算是基于持仓风险系数的动态估算，数据和诊断页是同一份 AES 加密持仓。」

## 亮点六：数据安全设计

| 设计决策 | 实现 | 用户价值 |
| -------- | ---- | -------- |
| 持仓不上云 | 仅存 `localStorage`，AES-256 加密 | 消除隐私顾虑 |
| 诊断最小化传输 | API 只收代码 + 用户填写价格 | 减少数据暴露面 |
| API Key 服务端化 | `.env` 注入，前端零接触 | 防止密钥泄露 |
| 一键清除 | `clearPortfolio()` 清持仓 + 诊断缓存 | 用户完全掌控 |

```typescript
// portfolioCrypto.ts
密钥派生: SHA256('lingxi-ai-finance-portfolio-v1')
算法:     AES-256 (CryptoJS)
存储格式: enc:v1:{ciphertext}
```

### 面试一句话

> 「持仓数据默认不出浏览器——AES-256 本地加密，诊断时只发送资产代码和用户自填价格。这是 PRD 阶段就定的隐私优先原则。」

## 亮点七：7 步审计 + 可解释溯源

每次诊断自动写入 7 步审计：

```
请求接收 → 数据获取 → 数据清洗 → LSTM v2.0 → RF v2.2 → 规则校验 → 结果生成
```

| 能力 | 说明 |
| ---- | ---- |
| 存储 | SQLite `audit_logs` 表 |
| 查询 | `GET /api/trace/{request_id}` |
| 可视化 | ECharts DAG 血缘图（`DataLineageChart.vue`） |
| 完整性 | `is_complete: len(logs) >= 7` |

### 面试一句话

> 「7 步不是文档写法，是和代码逐步对应的——面试官要验证我可以直接指到 `portfolio.py` 里每一行 `record_audit_step`。」
