## 第一幕：首页（0:00 ~ 0:30）

### 操作

1. 打开 `http://localhost:3015/`
2. 指向市场情绪简报（基于 Tushare 舆情数据实时计算）
3. 浏览 8 大功能入口

### 话术

> 「这是**灵析**——人机协同的 AI 理财辅助工具。四层架构：Vue 3 交互层、FastAPI 服务层、LSTM v2.0 + RF v2.2 + DeepSeek 模型层、Tushare + AKShare + AES 加密数据层。
>
> v1.4 重点：**七特征 LSTM（RMSE 3.39）**、**随机森林 v2.2（61% 准确率）**、诊断页追问助手、模型 HTTP 回滚、Docker/ECS 可部署。」

---

## 第二幕：导入持仓（0:30 ~ 1:00）

### 操作

1. 进入「导入持仓」
2. 展示已导入资产（茅台、平安银行、ETF）
3. 指向 AES-256 安全承诺

### 话术

> 「持仓通过 **AES-256 加密**写入 localStorage，**不上传云端**。保存后自动调 `/refresh-quotes` 从 Tushare **实时拉取**最新收盘。」

---

## 第三幕：资产周期 + LSTM 虚线（1:00 ~ 1:45）⭐

### 操作

1. 进入「资产周期分析」`/cycle`
2. 选择 **600519 贵州茅台**
3. **鼠标指向 PE 图末端橙色 LSTM 预测虚线**（未来 5 日）
4. 说明七特征输入

### 话术

> 「这是生产环境的 **LSTM v2.0**——输入 30 日 × **7 特征**：收盘价、涨跌幅、成交量、PE/PB 分位、换手率、振幅。估值数据 **AKShare 优先、Tushare 降级**。
>
> 在 HS300 300 股 × 3 年数据上，RMSE **3.39**，MAE **2.52**。
>
> 看这张图：蓝色实线是历史 PE，**橙色虚线是 LSTM 对未来 5 个交易日的预测延伸**。灰度 5% 流量可切新模型。」

---

## 第四幕：AI 持仓诊断（1:45 ~ 2:30）

### 操作

1. 进入「持仓诊断」
2. 点击 **「刷新现价」**（Tushare 实时拉价）
3. 指向风险资产 / 机会资产
4. 记住 `request_id`（溯源用）

### 话术

> 「诊断后端走 **7 步审计流水线**：请求接收 → 数据获取 → 数据清洗 → LSTM v2.0 → **随机森林 v2.2（14 维，61% 准确率）** → 规则引擎 → 结果生成。
>
> 接下来在页面**底部**介绍追问助手。」

---

## 第五幕：追问助手 — RAG + 合规（2:30 ~ 3:20）⭐ 高潮

### 操作

1. 滚动到诊断结果页 **底部**追问输入框
2. 输入：**「茅台现在能买吗」** → 展示拦截
3. 输入：**「我的持仓风险集中吗」** → 展示 AI 回答（带诊断上下文）
4. （可选）输入：**「宁德时代怎么样」**（不在持仓）→ 通用介绍

### 话术

> 「这是**诊断追问助手**，在诊断结果页底部。
>
> 第一个问题命中禁止词『能买吗』——**规则引擎前置拦截**，403，**零 LLM 调用**，写入 `compliance_logs`。
>
> 第二个问题合规放行——系统把刚才的诊断结果作为 **RAG 上下文**注入 Prompt，DeepSeek 基于真实持仓回答。
>
> 如果问持仓外的股票，走 external 模式，只做通用介绍，不给买卖建议。
>
> 这就是**合规闭环 + 上下文增强**。」

### 要点

- **语速放慢**，确保面试官看到拦截 UI
- 强调「持仓内带盈亏，持仓外通用介绍」

---

## 第六幕：分析溯源（3:20 ~ 4:00）

### 操作

1. 点击「分析溯源」→ `/trace/{requestId}`
2. **逐条展示 7 步审计日志**
3. 滚动到 ECharts **DAG 血缘图**

### 话术

> 「每次诊断有唯一 `request_id`，对应 **7 步审计日志**，持久化在 SQLite。
>
> DAG 图展示数据从 Tushare 进入 → 清洗 → LSTM + RF → 规则引擎 → 输出。回应『AI 黑盒』问题。」

---

## 第七幕：情绪纠偏（4:00 ~ 4:40）

### 操作

1. 进入「情绪纠偏」
2. 点击「刷新现价」
3. 指向四张行为偏差卡片：
   - **确认偏误** — 最高行业占比 %
   - **损失厌恶** — 亏损 X 只
   - **羊群效应** — 行业集中度
   - **行为矫正估算** — 基于真实持仓的动态估算

### 话术

> 「情绪指数基于 **Tushare 舆情数据实时计算**。
>
> **确认偏误**基于真实持仓行业集中度动态计算——比如白酒占比 60% 就高亮警告。
>
> **损失厌恶**统计亏损资产数量——来自成本价与 Tushare 现价的实时对比。
>
> **行为矫正估算**是基于真实持仓风险系数的**动态估算**——潜在损失、减少冲动交易次数、改善收益百分比。
>
> 数据和诊断页是同一份 AES 加密持仓。」

---

## 第八幕：投教聊天（4:40 ~ 5:10）

### 操作

1. 进入「投教」`/education`
2. 指向**右侧聊天面板**
3. 输入：**「什么是定投？」** 或 **「什么是 PE 估值？」**
4. 展示 DeepSeek 回答

### 话术

> 「投教页左边是结构化课程，**右边是 DeepSeek 即时问答**——用户学到概念马上可以追问。
>
> system prompt 严格限制：禁止买卖建议，200~400 字，带免责声明。」

---

## 第九幕：合规收尾 + 模型回滚（5:10 ~ 5:40）

### 操作（合规 20s）

1. 进入「合规说明」
2. 展示拦截统计（**基于系统运行日志的实时统计**）

### 操作（模型回滚 10s，可选）

3. 打开 Swagger 或终端调用：

```powershell
Invoke-RestMethod -Uri "http://localhost:8001/api/admin/model/rollback" `
  -Method POST -ContentType "application/json" `
  -Body '{"model_type":"lstm","target_version":"v1.0"}'
```

### 收尾话术

> 「总结：灵析 v1.4 = **LSTM v2.0（RMSE 3.39）× RF v2.2（61%）× 7 步审计 × 追问助手 RAG + 合规拦截 × 模型 HTTP 回滚热重载 × AES 本地加密 × 真实持仓纠偏 × Docker/ECS 部署**。
>
> 四层架构、Tushare 实时行情与历史缓存、Docker 可部署——这是一个可以审计、可以迭代、可以回滚的 AI 理财辅助工具。谢谢！」

## 应急方案

| 意外 | 应对 |
|------|------|
| Tushare 限流 | 「基于历史数据缓存动态更新」→ 点「刷新现价」触发实时拉取 |
| 周期图无虚线 | 换 600519 重试；需 30+ 日历史 |
| 诊断超时 | 展示 localStorage 缓存的上次诊断结果 |
| DeepSeek 未配置 | 跳过投教聊天和追问第二步，只展示合规拦截 |
| 后端未启动 | `curl localhost:8001/health` |
| 回滚不便执行 | 只展示 `GET /api/admin/model/versions` 列版本 |
