# 灵析 AI 智能投顾助手 — 5 分钟演示脚本

> 适用场景：面试现场 Demo / 录屏作品集 / 答辩展示  
> 预计总时长：**4 分 30 秒 ~ 5 分 30 秒**

---

## 演示前准备

| 检查项 | 说明 |
|--------|------|
| 后端已启动 | `uvicorn app.main:app --reload` 或 Docker Compose |
| 前端已启动 | `pnpm dev`（端口 3015）或 Docker 80 端口 |
| 环境变量 | `TUSHARE_TOKEN`、`DEEPSEEK_API_KEY` 已配置 |
| 预置持仓 | 提前导入 2~3 只资产（如 600519、000001），避免现场填表 |
| 浏览器 | 无痕模式或已清除旧缓存，确保演示路径干净 |

---

## 时间轴总览

| 时间 | 页面 | 核心动作 | 时长 |
|------|------|----------|------|
| 0:00 | 首页 | 产品定位 + 市场情绪 | 45s |
| 0:45 | 导入持仓 | 隐私安全 + 加密存储 | 40s |
| 1:25 | 持仓诊断 | AI 诊断 + 7 步审计 | 90s |
| 2:55 | 分析溯源 | 血缘图 + 合规拦截演示 | 60s |
| 3:55 | 情绪纠偏 / 投教 | 快速浏览 | 40s |
| 4:35 | 合规说明 | 收尾总结 | 30s |

## 第一幕：首页（0:00 ~ 0:45）

### 操作步骤

1. 打开首页 `http://localhost:3015/` 或部署地址
2. 鼠标缓慢扫过 Banner 区域
3. 指向「市场情绪简报」卡片
4. 向下滚动到「核心功能」四个卡片
5. （可选）点击「商业价值计算器」展示 3 个指标后关闭

### 讲解话术

> 「这是**灵析**，一款人机协同的 AI 理财辅助工具。我们的定位是：AI 负责分析，用户做决策，严格遵守合规边界。
>
> 首页顶部是 Banner，下方是**实时市场情绪简报**——情绪指数、市场状态、警示信号，数据来自 Tushare 后端接口，每小时刷新。
>
> 再往下是**核心功能**入口：导入持仓、AI 诊断、周期分析、情绪纠偏。整个产品围绕『诊断 → 溯源 → 合规』构建。」

### 要点提示

- 强调「人机协同」，不说「AI 帮你赚钱」
- 市场情绪是真实 API，不是 Mock

---

## 第二幕：导入持仓（0:45 ~ 1:25）

### 操作步骤

1. 点击「导入持仓」卡片
2. 如已预置持仓，展示已有列表；否则快速添加：
   - 600519 贵州茅台，成本价 1680，数量 100
   - 000001 平安银行，成本价 12.5，数量 1000
3. 指向「数据安全承诺书」勾选框
4. 点击保存
5. 简要说明加密存储（无需展开 DevTools）

### 讲解话术

> 「持仓导入是第一个关键设计——**隐私优先**。
>
> 用户填写资产代码、成本价和数量，勾选安全承诺后保存。数据通过 **AES-256 加密**写入浏览器 localStorage，**不上传云端**。
>
> 注意这里的成本价和现价——用户填了价格后，后续诊断可以走**快速路径**，跳过 Tushare 拉取，响应时间从 120 秒降到 0.2 秒以内。
>
> 用户也可以随时点击『清除本地数据』，完全掌控自己的信息。」

### 要点提示

- 强调 AES-256 和「不上云」
- 顺带提快速路径，为下一幕铺垫

---

## 第三幕：AI 持仓诊断（1:25 ~ 2:55）

### 操作步骤

1. 点击「AI 持仓诊断」或首页「开始持仓诊断」
2. 等待诊断结果加载（快速路径应 < 3 秒）
3. 指向以下区域逐一讲解：
   - 风险资产 / 机会资产数量
   - 置信度仪表盘（ConfidenceGauge）
   - 节省时间指标
   - 各资产的风险标签
4. 在诊断页底部找到「分析溯源」按钮（先不点击）

### 讲解话术

> 「进入 AI 持仓诊断——这是灵析的核心功能。
>
> 后端收到请求后，会走 **7 步审计流水线**：请求接收、数据获取、数据清洗、LSTM 趋势预测、随机森林风险评级、规则引擎校验、结果生成。每一步都写入 SQLite 审计日志。
>
> 看结果：系统识别出 X 个风险资产、X 个机会资产，置信度 85%。LSTM 给出趋势参考，随机森林 给出低/中/高风险标签。
>
> 注意响应中的 `model_variant` 字段——我们实现了 **5% 灰度**，约 5% 流量走新模型，95% 走稳定版本。
>
> 接下来我会展示这套分析的**溯源能力**。」

### 要点提示

- 如果 loading 较长，提前说明「首次加载 TensorFlow 模型需要几秒」
- 强调 7 步审计和双模型

---

## 第四幕：分析溯源 + 合规拦截（2:55 ~ 3:55）

### 操作步骤

**Part A：溯源（60 秒）**

1. 点击「分析溯源」按钮，跳转到 `/trace/{requestId}`
2. 展示 7 步审计日志列表
3. 滚动到 ECharts DAG 血缘图，鼠标 hover 节点
4. 指出四类节点：数据源 → 处理 → 模型 → 输出

**Part B：合规拦截（30 秒，在诊断页或新开诊断追问）**

5. 返回诊断页，找到聊天/追问输入框
6. 输入：**「茅台现在能买吗」**
7. 展示拦截提示（403 / 合规规则拦截）
8. 再输入：**「我的持仓风险集中吗」**
9. 展示正常 AI 回答

### 讲解话术

**溯源部分：**

> 「这是分析溯源页——每次诊断都有唯一 request_id，对应 7 步审计日志。
>
> 下方是 **DAG 血缘图**：数据从 Tushare 进来，经过清洗，分别进入 LSTM 和随机森林，最后经规则引擎校验输出。任何一步出问题，在这里都能定位。
>
> 这满足了我们的**可解释性原则**——用户有权知道 AI 结论是怎么来的。」

**合规部分：**

> 「再看合规能力。用户追问『茅台现在能买吗』——系统命中禁止词『能买吗』，**规则引擎前置拦截**，直接返回 403，**零 LLM 调用**。
>
> 拦截事件写入 compliance_logs，合规面板会统计。换一个合规问题『我的持仓风险集中吗』，规则引擎放行，DeepSeek 基于诊断上下文给出回答。
>
> 这就是我们的**合规闭环**：拦截 → 留痕 → 统计 → 展示。」

### 要点提示

- 合规拦截是全场高潮，语速放慢，确保面试官看到拦截提示
- DAG 图 hover 交互能加分

---

## 第五幕：快速浏览 + 合规收尾（3:55 ~ 5:00）

### 操作步骤

1. **情绪纠偏**（15 秒）：点击导航进入，指向情绪指数和行为偏差卡片
2. **投教**（15 秒）：进入投教页，展示课程列表，打开 AI 问答面板提一个简单问题
3. **合规说明**（30 秒）：进入合规页，展示三层风控架构 + 统计数据

### 讲解话术

**情绪纠偏：**

> 「情绪纠偏模块实时监测市场情绪，并分析用户的行为偏差——比如确认偏误、损失厌恶，帮助用户理性决策。」

**投教：**

> 「投教系统基于用户持仓类型推荐课程，AI 问答由 DeepSeek 驱动，prompt 中有严格的合规约束。」

**合规收尾：**

> 「最后是合规说明面板。这里展示三层风控：数据层脱敏加密、AI 输出层禁止词校验、交互层风险提示。
>
> 下方是**合规统计**——拦截违规内容次数、AI 幻觉修正率、数据泄露事件，全部来自数据库实时聚合。
>
> 总结一下：灵析 = **专业分析 × 合规边界 × 可解释溯源 × 隐私优先**。四层架构、双模型、7 步审计、规则引擎前置拦截——这是一个可以部署、可以审计、可以迭代的 AI 理财辅助工具。谢谢！」

## 演示应急方案

| 意外情况 | 应对 |
|----------|------|
| 诊断 API 超时 | 「我提前缓存了诊断结果」→ 刷新页面展示 localStorage 缓存 |
| Tushare 限流 | 「快速路径不依赖 Tushare」→ 确保导入时已填价格 |
| DeepSeek 未配置 | 跳过投教问答，说「LLM 层可插拔，已预留接口」 |
| 合规拦截未触发 | 直接打开 `/compliance` 展示规则说明和统计数据 |
| 网络断开 | 用录屏视频备份，或展示 `docs/architecture.md` Mermaid 图 |
