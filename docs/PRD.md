## 1. 项目背景与用户痛点

### 1.1 行业背景

中国个人投资者规模持续扩大，但多数散户缺乏系统的持仓分析工具，容易在信息过载、市场波动和情绪化交易之间做出非理性决策。与此同时，监管对 AI 理财辅助工具的要求日益严格：**不得荐股、不得承诺收益、必须可解释、必须留痕**。

### 1.2 目标用户

| 用户类型 | 典型画像 | 核心诉求 |
|----------|----------|----------|
| 初级投资者 | 持有 3~10 只基金/股票，缺乏分析框架 | 看懂持仓风险，学习基础理财知识 |
| 进阶投资者 | 有一定经验，关注周期与情绪管理 | 辅助判断估值区间，避免追涨杀跌 |
| 产品/合规评审方 | 关注 AI 边界与风控能力 | 验证系统是否合规、可审计、可落地 |

### 1.3 用户痛点

| 痛点 | 具体表现 | 灵析的应对方向 |
|------|----------|----------------|
| **看不懂持仓** | 不知道集中度过高、亏损资产占比 | AI 持仓诊断 + 风险分级 |
| **不知道什么时候买/卖** | 频繁问「能买吗」「什么时候卖」 | 明确 AI 边界，拒绝买卖建议，改做周期参考 |
| **情绪化交易** | 恐慌抛售、追涨冲动 | 市场情绪监测 + 行为偏差提示 |
| **缺乏投教** | 知识碎片化，与自身持仓无关 | 场景化投教，基于持仓类型推荐课程 |
| **不信任 AI 结论** | 不知道数据从哪来 | 7 步审计日志 + 溯源血缘图 |
| **隐私顾虑** | 不愿上传持仓到云端 | AES-256 本地加密存储，持仓默认不上云 |

## 2. 产品定位与核心价值

### 2.1 一句话定位

**灵析**是一款**人机协同的 AI 理财辅助工具**——AI 负责数据处理与信息提炼，用户做最终决策；严格遵守 AI 能力边界，确保合规合法。

### 2.2 核心价值主张

```
专业分析  ×  合规边界  ×  可解释溯源  ×  隐私优先
```

| 价值维度 | 说明 |
|----------|------|
| **专业分析** | LSTM 周期预测 + 随机森林风险评级 + 行情数据融合 |
| **合规边界** | 规则引擎拦截禁止词，三层风控兜底 |
| **可解释溯源** | 每次诊断 7 步审计，支持 DAG 血缘可视化 |
| **隐私优先** | 持仓 AES-256 本地加密，用户可随时清除 |

### 2.3 产品边界（AI 能做什么 / 不能做什么）

| ✅ AI 能做 | ❌ AI 不能做 |
|-----------|-------------|
| 分析持仓结构与风险分布 | 推荐具体买卖时机 |
| 判断资产估值周期阶段（参考） | 预测股价/承诺收益 |
| 监测市场情绪与行为偏差 | 提供内幕消息或荐股 |
| 生成投教内容与知识解答 | 替代用户做投资决策 |
| 展示分析依据与数据来源 | 保证分析结果 100% 准确 |

### 2.4 四大设计原则

1. **人机协同原则**：AI 负责数据处理，用户做最终决策  
2. **AI 边界管控原则**：明确能力边界，禁止越界输出  
3. **可解释性原则**：所有结论附带数据来源、分析维度、规则校验  
4. **分层风控原则**：数据层、AI 输出层、交互层三重防护

## 3. 功能架构（四层架构）

灵析采用**工业级四层架构**，从用户交互到数据存储分层解耦：

```
┌─────────────────────────────────────────────────────────┐
│  第一层：交互层（Presentation Layer）                      │
│  Vue 3 前端 · 7 大功能模块 · 风险提示 · 免责声明            │
├─────────────────────────────────────────────────────────┤
│  第二层：服务层（Service Layer）                         │
│  FastAPI · 规则引擎 · 审计日志 · 溯源服务 · 合规统计       │
├─────────────────────────────────────────────────────────┤
│  第三层：模型层（Model Layer）                            │
│  LSTM 周期预测 · 随机森林风险评级 · DeepSeek 投教问答       │
├─────────────────────────────────────────────────────────┤
│  第四层：数据层（Data Layer）                             │
│  Tushare 行情 · SQLite 审计库 · 本地加密持仓 · 知识库 JSON  │
└─────────────────────────────────────────────────────────┘
```

### 3.1 模块地图

| 模块 | 路由 | 后端 API | 依赖层 |
|------|------|----------|--------|
| 导入持仓 | `/portfolio/import` | — | 交互层 + 本地加密 |
| AI 持仓诊断 | `/portfolio/diagnosis` | `POST /api/portfolio/diagnose` | 服务层 + LSTM + RF + 规则引擎 |
| 资产周期分析 | `/cycle` | `POST /api/cycle/analyze` | 服务层 + LSTM + Tushare |
| 情绪纠偏 | `/emotion` | `GET /api/market/sentiment` | 服务层 + Tushare |
| 场景化投教 | `/education` | `POST /api/education/chat` | 服务层 + DeepSeek |
| 分析溯源 | `/trace/:requestId` | `GET /api/trace/{id}` | 服务层 + SQLite 审计 |
| 合规说明 | `/compliance` | `GET /api/admin/compliance-stats` | 服务层 + 合规日志 |
| 诊断追问 | 诊断页内嵌 | `POST /api/diagnose/chat` | 规则引擎 + DeepSeek |

### 3.2 诊断流水线（7 步审计）

持仓诊断是核心链路，服务层按以下步骤编排并写入 `audit_logs`：

1. 请求接收  
2. 数据获取（Tushare / 用户本地价格）  
3. 数据清洗（异常值拦截）  
4. LSTM 模型预测（趋势参考）  
5. 随机森林风险评估（低/中/高）  
6. 规则引擎校验（禁止词 / PE 异常）  
7. 结果生成

## 4. 功能优先级（MoSCoW）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **Must** | 导入持仓 + 本地加密 | 隐私承诺的基础，无此无信任 |
| **Must** | AI 持仓诊断 | 核心差异化能力 |
| **Must** | 规则引擎 + 合规拦截 | 监管合规底线 |
| **Must** | 分析溯源（审计日志） | 可解释性要求 |
| **Should** | 资产周期分析（LSTM） | 增强专业度 |
| **Should** | 情绪纠偏 | 差异化用户价值 |
| **Should** | 场景化投教 | 提升留存与教育价值 |
| **Could** | 模型灰度发布 | 工程化能力展示 |
| **Could** | 商业价值计算器 | 量化 ROI 叙事 |
| **Won't（v1）** | 用户账号体系 / 云端同步持仓 | 隐私优先，暂不做 |
| **Won't（v1）** | 实时荐股 / 买卖信号推送 | 违反 AI 边界 |

## 5. 核心用户旅程

### 旅程 A：新用户首次诊断（主路径）

```
打开首页
  → 查看市场情绪简报（建立信任）
  → 点击「开始持仓诊断」
  → 导入持仓（填写代码/成本/数量，勾选安全承诺）
  → AES 加密写入 localStorage
  → 进入诊断页，触发 POST /api/portfolio/diagnose
  → 查看风险资产 / 机会资产 / 置信度
  → 点击「分析溯源」查看 7 步审计 + 血缘图
  → （可选）在诊断页追问 AI，规则引擎前置校验
```

**关键触点**：安全承诺弹窗、诊断 loading、溯源 DAG 图、合规拦截提示

### 旅程 B：情绪纠偏

```
进入情绪纠偏页
  → 拉取 /api/market/sentiment（情绪指数、市场状态、警示信号）
  → 查看行为偏差分析（确认偏误、损失厌恶等）
  → 基于本地持仓的行为矫正估算
  → 阅读理性引导建议
```

### 旅程 C：投教学习

```
进入投教页
  → 浏览课程列表（基于持仓类型推荐）
  → 进入课程详情
  → 打开 AI 问答面板，POST /api/education/chat
  → DeepSeek 生成个性化解答（合规约束 prompt）
```

### 旅程 D：合规审查（B 端视角）

```
进入合规说明页
  → 查看三层风控架构说明
  → 拉取 /api/admin/compliance-stats（拦截次数、幻觉修正率等）
  → 查看禁止词库与 AI 能力边界声明
```

## 6. 商业价值测算

### 6.1 价值假设

| 假设 | 依据 |
|------|------|
| 单用户年分析次数 ~30 次 | 月均 2~3 次主动复盘 |
| 每次人工分析耗时 ~30 分钟 | 查阅财报、行情、新闻 |
| AI 辅助单次节省 ~25 分钟 | 自动聚合数据 + 结构化输出 |
| 情绪化交易年均潜在损失 ~5,000~10,000 元 | 行业行为金融学研究区间 |

### 6.2 产品内计算器（演示模型）

首页「商业价值计算器」基于以下公式（演示值）：

| 指标 | 计算逻辑 | 演示输出 |
|------|----------|----------|
| 节省分析时间 | 30 次/年 × 25 分钟 ≈ 12.5 小时，× 效率系数 | **156 小时/年** |
| 延长持有优质资产 | 减少频繁换仓，估算额外持有天数 | **23 天** |
| 避免情绪化损失 | 情绪纠偏 + 规则拦截的综合效益 | **8,500 元/年** |

> 注：以上为产品演示模型，面试中应说明「基于假设的参数化估算，非真实 A/B 实验数据」。

### 6.3 合规价值量化

| 指标 | 数据来源 | 意义 |
|------|----------|------|
| 拦截违规内容次数 | `compliance_logs` 表聚合 | 证明规则引擎真实运行 |
| AI 幻觉修正率 | 审计日志 + 人工反馈 | 模型输出质量监控 |
| 数据泄露事件 | 本地加密 + 不上云设计 | 隐私合规背书 |
| 规则校验通过率 | 诊断流水线统计 | 系统稳定性指标 |

### 6.4 ROI 叙事模板（面试用）

> 「以一名持有 8 只资产的散户为例：灵析将其单次持仓分析从 30 分钟压缩到 2 分钟以内，年化节省约 150+ 小时；同时通过情绪纠偏模块和行为偏差提示，帮助用户减少 8~12 次冲动交易，估算避免损失 5,000~10,000 元。合规层面，规则引擎已拦截 X 次违规提问，审计日志覆盖 100% 诊断请求。」

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 诊断 API 快速路径（本地价格）< 1s；含 Tushare 拉取 < 30s |
| 安全 | 持仓 AES-256 本地加密；API Key 仅存服务端 `.env` |
| 合规 | 禁止词库可配置；每次 AI 输出带免责声明 |
| 可观测 | 7 步审计日志；`model_variant` 灰度标记 |
| 部署 | Docker Compose 一键部署；秒悟云平台 `.env.cloud` 模板 |
